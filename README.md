# Frontend Projects Monorepo

프론트엔드 개발자로 성장하기 위한 **개인 프로젝트 및 포트폴리오 모노레포**  
기본기 → 구조 → 설계 → 확장성을 기준으로 프로젝트를 관리합니다.

---

## 🎯 Repository Purpose

- HTML / CSS / JS / React 기반 프로젝트를 **결과물과 구조 중심으로 기록**
- “동작하는 코드”보다 **읽고 이해 가능한 구조**를 우선
- 퍼블리셔 → 프론트엔드 개발자로 이어지는 **성장 과정 증명**

> 면접이나 포트폴리오 리뷰에서 바로 보여줄 수 있는 상태를 목표로 합니다.  
> 프로젝트와 구조는 **꾸준히 개선·업데이트**됩니다.

---

## 🧠 How I Think

1. **구조 중심**
   - 시맨틱 마크업, 역할 단위 스타일, JS는 보조 수단
2. **재사용 전제**
   - 컴포넌트 단위 사고, 확장 가능한 클래스/스타일
3. **실무 기준**
   - 모바일 우선, 접근성, 성능 최적화
